# 非機能要件 (まとめ)

セキュリティ・テーマ・データ構造の概略要件をまとめる。詳細設計は各設計書を参照。

## セキュリティ

- **認証**: 将来的にメール+パスワード認証を導入。パスワードはハッシュ化して保存。可能なら 2FA を提供。
- **アクセス制御**: ユーザーロールに応じた権限管理。リソースごとにアクセス許可を制御。
- **データ保護**: 機微情報は暗号化保存。通信は SSL/TLS 必須。詳細は `docs/ja/develop/design/data/data-security.md`。
- **ログ**: 重要イベント（ログイン・データ変更等）をログ化し定期監査。
- **脆弱性管理**: 定期セキュリティテスト、依存ライブラリの脆弱性ウォッチ。
- **インシデント対応**: 手順を策定し、発生時に迅速対応。

## テーマ

カスタマイズ機能の一環として 3 種のテーマを提供する。

- **公式テーマ**: アプリ組み込み（ライト・ダーク等）。ネット接続不要。
- **公開テーマ**: ユーザー作成テーマをテーマストアで公開・配布。公開には登録要、ダウンロードは登録不要。ログイン時はお気に入り等の付加機能あり。
- **ユーザーローカルテーマ**: ローカル動作。CSS で定義。共有時はファイル送付（ネット接続不要）。

CSS をベースに、必要に応じて JSON 表現も検討する。

## データ構造のグループ

データは 3 つのグループに分かれる。詳細は `docs/ja/develop/design/data/data-model.md` を参照。

- **ユーザー設定**: JSON ローカル保存。必要に応じてサーバ同期。
- **プロジェクト**: Automerge + SQLite + PostgreSQL（サーバ）。プログラムからは SQLite を参照（Automerge を SQLite に反映するパイプライン）。Automerge のドキュメントはプロジェクト単位、ID キーは不要。
- **ユーザー一覧**: Automerge + SQLite。プロジェクト同様、SQLite 参照。
