# 日時フォーマット設定ページ

設定画面で日時フォーマットをカスタマイズする UI の仕様。

> 実装の正本: `src/lib/types/datetime-format.ts` (ドメイン型) / `src/lib/stores/datetime-format.svelte.ts` (ストア状態と永続化)。

## 適用先

- 日時入力コンポーネント全般
- 日時表示コンポーネント全般

## グループ (group enum)

4 種類:

1. **デフォルト** (空文字): 常に 1 つ。アプリ定義、変更不可
2. **プリセット**: アプリ定義、変更不可 (国・地域別の標準設定)
3. **カスタム** (空): 常に 1 つ。ユーザー入力済み
4. **カスタムフォーマット**: ユーザーが定義したフォーマット

## ID 体系

| 種別 | ID 形式 | 永続化 |
| --- | --- | --- |
| アプリ固定 (デフォルト・プリセット) | 負の整数 (例: `-1`, `-2`) | DB に保存しない (TypeScript 内で定義) |
| ユーザーカスタムフォーマット | UUID | DB に保存 |

## アプリ提供データ

- **デフォルト**: `id=-1`, `name`=言語に応じた「デフォルト」相当, `format=""`, `group=デフォルト`
- **プリセット**: 例 `id=jp-0` (日本・西暦・24h `yyyy年MM月dd日 HH:mm:ss`), `id=jp-1` (和暦・12h), `id=en-1` (America `MM/dd/yyyy HH:mm:ss`) 等
- **カスタム**: `id=-2`, `name`=言語に応じた「カスタム」相当, `format=""`, `group=カスタム`

## UI コンポーネント

| 名称 | 役割 |
| --- | --- |
| テスト日時入力 | プレビュー用の日時入力。初期表示時は現在日時、以後ユーザー自由入力 |
| 日時フォーマット (テキストボックス) | アプリで使うフォーマット。設定値ストアと常時同期 |
| 日時フォーマットプレビュー | テスト日時入力 + 日時フォーマットで日時表示 |
| ↓ ボタン | 日時フォーマット → テストフォーマットへ反映 |
| ↑ ボタン | テストフォーマット → 日時フォーマットへ反映 |
| テストフォーマット (テキストボックス) | 試行用 (ストアと独立) |
| テストフォーマットプレビュー | テスト日時入力 + テストフォーマットで日時表示 |
| テストフォーマット選択 (コンボボックス) | テストフォーマットと連動。選択変更でテストフォーマットへ反映 |
| フォーマット名変更前ラベル | カスタムフォーマット選択時のみ表示 |
| フォーマット名 (テキストボックス) | カスタム/カスタムフォーマット選択時のみ表示。名前付け/変更 |
| フォーマット追加 (ボタン) | カスタム/カスタムフォーマット選択時のみ。新規追加 |
| フォーマット上書き (ボタン) | カスタムフォーマット選択時のみ。既存を上書き保存 |
| フォーマット削除 (ボタン) | カスタムフォーマット選択時のみ。確認ダイアログ後に削除 |

## イベント処理

| イベント | 処理 |
| --- | --- |
| 初期表示 | 日時フォーマットの内容をテストフォーマットにコピー (同期はしない)。これに伴い「テストフォーマット選択」「フォーマット名」「フォーマット保存」の表示状態が変化 |
| 日時フォーマット入力変更 | ストアを更新 |
| ↓ ボタン | 日時フォーマット → テストフォーマット反映、関連表示状態が変化 |
| ↑ ボタン | テストフォーマット → 日時フォーマット反映 |
| テストフォーマット入力変更 | format が一致するものがあれば「テストフォーマット選択」を自動選択。関連表示状態が変化 |
| テストフォーマット選択変更 | 「カスタム」選択時は何もしない。それ以外は入力内容に応じて状態変化 |
| フォーマット名変更 | 特になし |
| フォーマット追加 | UUID で新規追加 (衝突時は最大 10 回再割当) |
| フォーマット上書き | id 一致のフォーマットを直接書換 |

## 状態管理 (UI 設計)

### ストア管理 (即座反映)

- `currentFormat`: 現在の日時フォーマット (設定値)
- `customFormats`: ユーザー定義カスタムフォーマット一覧
- `allFormats`: アプリ固定 + カスタムの統合リスト (`$derived`)

### 親コンポーネント管理 (ローカル状態)

- `testDateTime`: テスト用日時入力
- `testFormat`: テストフォーマット (試行用、ストアと独立)
- `testFormatName`: フォーマット名入力
- `selectedPreset`: 選択中のプリセット (`$derived` で `testFormat` から自動算出)

### 反映ルール

- 日時フォーマット変更 → 即座にストア反映 (電源断対策)
- カスタムフォーマット操作 → 即座にストア反映
- テストフォーマット変更 → `selectedPreset` が `$derived` で自動更新
- 初期化: `$effect` でストアから `testFormat` を初期化

## 注意点

- イベント処理が破綻しないよう Svelte 5 runes (`$state`/`$derived`/`$effect`) で表現する

### バリデーション方針

- 日時フォーマット文字列の妥当性チェックはユーザー責任。アプリ側ではバリデーションを行わない
- 不正フォーマットでもそのまま保存・表示し、実際の日時変換エラー時に適切にハンドリングする

### 国際化対応

- プリセット名や UI 表示の多言語対応は、ストア内の言語設定をベース
- アプリ固定データ (デフォルト・プリセット) の表示名は、プログラム内で言語設定に応じて動的生成
