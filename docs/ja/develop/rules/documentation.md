# ドキュメント編集ルール

## 命名規則

- 基本はハイフン区切り (kebab-case)。
- 例外: ファイル名が DB のテーブル名等になっている場合は、その実物名に合わせる。
- スペースが含まれる場合はハイフンまたはアンダースコアに置換して保存する（スペースのまま保存しない）。

## 多言語対応

- `docs/ja/` (日本語版) と `docs/en/` (英語版) の構造・内容は一致させる。
- 通常時は両方を同時に更新する。片方のみの更新は禁止。
- 機械翻訳に頼らず適切な翻訳を行う。
- ※ 例外: 大規模な再構成タスクでは ja を先行し、en は別タスクで追従させる場合がある。skill `documentation/SKILL.md` の方針を参照。

## 編集手順

1. 日本語版を編集
2. 英語版を対応する内容で編集（同期タスクで実施する場合あり）
3. 両者が一致していることを確認
4. 関連ドキュメントへの参照リンクが切れていないか確認
